01

PROJECT OVERVIEW

Signal Bloom is an experimental audiovisual performance system that combines live coding, generative art, artificial intelligence, and improvisational performance into a single evolving environment.

Signal Bloom treats generation as a performative event.

  • INPUT_00AUDIO / LIVE
  • INPUT_01PERFORMER / LIVE CODING + IMPROV
  • INPUT_02AUDIENCE / ENERGY + INTERACTION
  • PIPE_00HYDRA → COMFYUI → ROUTER
  • OUT_00LIVE VISUAL OUTPUT / PROJECTION
02

THESIS

Signal Bloom explores how improvisation between human and machine actors creates conditions for emergence.

The project investigates the evolving relationship between:

  • PERFORMER
  • MUSICIAN
  • MACHINE
  • GENERATIVE SYSTEMS
  • AUDIENCE

The resulting experience is shaped through distributed authorship, where creative influence is shared across human, computational, and environmental participants.

03

THE QUESTION

AI-generated visuals are often treated as finished assets: generated, rendered, imported, and played back.

> audio.listen() // live > generate(prompt) // streaming > hydra.modulate(o0) // live > generation := performance // TRUE

What happens when generation becomes the performance?

04

ARTISTIC VISION

Signal Bloom exists at the intersection of disciplines. The visual language draws inspiration from signal processing, cybernetic systems, digital emergence, recursive structures, iridescent materials, and computational organisms.

The visual world feels synthetic, organic, unstable, and alive.

  • DIGITAL DECAY
  • ATMOSPHERIC GLOW
  • DATAMOSH COMPRESSION
  • QUEER FUTURISM
  • GLITCH AESTHETICS
  • CHROMATIC ABERRATION
  • SIGNAL CORRUPTION
05

NARRATIVE FRAMEWORK

ORPHEUS / EURYDICE — A MYTHIC FRAMEWORK FOR TRANSFORMATION. EACH MOVEMENT BINDS TO A LIVE HYDRA SKETCH.

06

IMPROVISATION & EMERGENCE

IMPROVISATION the live decisions made by humans and systems.
EMERGENCE the visual behavior that arises from those decisions.
  • SOUND CHANGES THE IMAGE.
  • CODE CHANGES THE SYSTEM.
  • AI INTRODUCES VARIATION.
  • THE AUDIENCE CHANGES THE ROOM.

Through these interactions, emergence becomes possible.

All image generation occurs locally using open-source tools, custom-trained visual models, and artist-directed workflows. The system is designed as a performance instrument rather than an automated content generator.

08

CRYSTALLIZATION ARC

Signal Bloom exposes the image generation process itself.

NOISE→STRUCTURE→FORM→IMAGE

Intermediate denoising frames are streamed directly into the visual environment. The audience witnesses emergence as it occurs.

The audience experiences the image through its transformation from noise to form.

10

OPEN SYSTEMS

Signal Bloom is built on open-source tools, shared knowledge, and community-driven software.

Hydra, p5.js, Three.js, ComfyUI, and Stable Diffusion exist because artists, developers, and researchers chose to make their work available to others.

In that spirit, Signal Bloom is being developed as an open system. The software provides a framework, but the artistry emerges through performance, improvisation, and human decision-making.

The most interesting outcomes are often the ones no individual could have planned.

FINAL STATEMENT

Signal Bloom investigates the space between intention and emergence.

Through improvisation, performers and machines shape an evolving visual environment that can never be fully predicted in advance.

Each performance becomes an experiment in collaboration, uncertainty, and transformation.
