PROJECT OVERVIEW

Signal Bloom is an experimental audiovisual performance system that combines live coding, generative art, artificial intelligence, and improvisational performance into a single evolving environment.

The project explores how performers and machines can collaborate to create emergent visual experiences in real time.

Rather than relying on pre-rendered media, Signal Bloom transforms the act of generation itself into a performative event. Images, forms, textures, and visual structures emerge live during performance through interactions between code, sound, AI systems, and human improvisation.

⸻

THESIS

Signal Bloom explores how improvisation between human and machine actors creates conditions for emergence.

The project investigates the evolving relationship between:

* Performer
* Musician
* Machine
* Generative Systems
* Audience

Through live interaction, these participants contribute to experiences that cannot be fully predicted or authored by any single actor.

⸻

PROBLEM STATEMENT

Most AI-generated visual media is created before a performance begins.

Images are generated.
Videos are rendered.
Assets are imported.

The performance becomes a playback system.

Signal Bloom asks a different question:

What happens when generation itself becomes the performance?

ARTISTIC VISION

Signal Bloom exists at the intersection of:

* Live Coding
* Generative Art
* AI-Assisted Creativity
* Queer Futurism
* Glitch Aesthetics
* Immersive Media
* Experimental Performance

The visual language draws inspiration from signal processing, cybernetic systems, digital emergence, recursive structures, iridescent materials, and computational organisms.

The work seeks to balance technological and organic forms, creating environments that feel simultaneously synthetic and alive.

IMPROVISATION AND EMERGENCE

One of the central discoveries of the project is the distinction between improvisation and emergence.

Improvisation describes the actions.

Emergence describes the outcomes.

The performer improvises.

The musician improvises.

The generative systems improvise.

The AI system introduces variation and uncertainty.

Through these interactions, emergence becomes possible.

Signal Bloom explores how improvisation creates conditions for emergence.

DESIGN SPECIFICATION

CORE PRINCIPLE

The website should feel like an artifact generated by Signal Bloom itself.

It should not feel polished.

It should not feel corporate.

It should not feel modern.

It should feel like a machine attempting to document itself.

Glitch is not decoration.

Glitch is the argument.

⸻

COLOR SYSTEM

Background:
#000000

Primary Signal:
#00f0ff

Corruption:
#ff00ee

Depth:
#1a1aff

Alert:
#ffaa00

Overlay:
rgba(0,0,0,0.85)

⸻

TYPOGRAPHY

Only monospace fonts.

Preferred:

‘Courier New’, Courier, monospace

Alternatives:

* JetBrains Mono
* IBM Plex Mono

Never use:

* Sans-serif fonts
* Serif fonts
* Font icons

Never use bold.

Use color and scale for hierarchy.

All labels should be uppercase.

⸻

LAYOUT

No cards.

No rounded corners above 2px.

No soft UI.

No physical depth.

Thin borders only.

High information density.

Asymmetry is encouraged.

Metadata belongs on edges.

Content belongs in the center.

⸻

HERO SECTION

Use one of:

1. Live video capture from Signal Bloom
2. Recorded tunnel footage
3. Full-screen system screenshot

Never use stock imagery.

All imagery should originate from the system itself.

⸻

CODE AS VISUAL MATERIAL

Display actual Hydra code throughout the site.

Code should function as both documentation and texture.

Large code blocks may be layered over imagery using:

mix-blend-mode: difference

Line numbers should appear dimmed.

Selected lines may be highlighted.

⸻

MOTION

Motion should feel unstable.

Transitions:

0.2s–0.3s linear

No bounce.

No parallax.

No polished easing curves.

Use occasional glitch displacement on headings.

⸻

GLOBAL OVERLAY

Apply scanlines across the entire site.

The scanline layer should unify all content under a single visual register.

⸻

FINAL STATEMENT

Signal Bloom is not simply a software project.

It is an investigation into what happens when performers, machines, and generative systems improvise together.

The goal is not to create visuals.

The goal is to create conditions for emergence.

PRIMARY DIRECTIVE:
Build this as a long-form single-page site using Vite. Create sections matching the markdown structure. Use the Design Specification as a strict visual system. Prioritize atmosphere, readability, and immersion over traditional portfolio conventions. The site should feel like a living technical artifact from Signal Bloom rather than a polished marketing page.
